**시행일**: (게시 시 확정) · **버전**: 0.1

NEXUS("앱")는 당신의 움직임을 캐릭터의 성장으로 바꾸는 게임입니다. 우리는 데이터를 최소한으로
다루며, 이 문서는 앱이 실제로 무엇을 읽고, 무엇을 저장하고, 무엇을 절대 보내지 않는지 설명합니다.

## 1. 핵심 요약

- 건강 데이터(걸음·운동·심박·수면)는 **기기 안에서만** 읽고 계산합니다. **원본 수치는 기기 밖으로
  나가지 않습니다.**
- 서버가 없습니다. 계정도 없습니다. 광고가 없고, 데이터를 판매하지 않으며, 제3자와 공유하지 않습니다.
- 앱이 기기 밖으로 보낼 수 있는 것은 두 가지뿐입니다: ① 익명 사용 통계(무엇이 "일어났다"는 사실만,
  수치 없음) ② 앱 오류 보고(크래시 정보 — 건강 수치 없음, 숫자 자동 마스킹).
- 설정 → 모든 데이터 삭제로 언제든 앱의 모든 기록을 지울 수 있습니다.

## 2. 읽는 것 — Health Connect (기기 내 처리)

앱은 Android Health Connect에서 다음을 **읽기 전용**으로 가져옵니다:

| 데이터 | 용도 |
|---|---|
| 걸음 수 | 걷기 성장(XP) 계산 |
| 운동 세션(유형·시간) | 운동 성장(XP) 계산 |
| 심박수 | 운동 강도 보정 |
| 수면 | 컨디션 회복 보정 |
| 백그라운드 읽기 | 앱을 켜지 않아도 성장 반영(15분+ 주기) |
| 과거 이력 | 최초 연결 시 지난 기록을 소급해 시작 레벨 계산 |

읽은 원본 수치는 **저장하지 않습니다**. 기기 안에서 XP·컨디션 같은 게임 수치로 계산한 뒤 그 결과만
앱 내부에 기록합니다. 원본은 Health Connect가 관리하며, 권한은 시스템 설정에서 언제든 회수할 수
있습니다(회수해도 앱은 데모 모드로 동작합니다).

## 3. 저장하는 것 — 이 기기 안

- **성장 원장**: 계산된 XP 값, 계산 공식 버전, 발생 시각, 데이터 출처 앱 이름(예: 삼성헬스),
  기록 방식(자동/수동), Health Connect 기록 식별자(무작위 ID — 중복 지급 방지용)
- **게임 상태**: 레벨·컨디션·에너지·원정 진행·배지, 설정(주간 목표·휴식 모드·알림)

## 4. 백업 — 당신의 통제 아래

- **수동 백업**: 설정에서 성장 기록을 JSON 파일로 내보내고 가져올 수 있습니다. 파일 저장 위치는
  당신이 고르며, 파일에는 원본 건강 수치가 없습니다(§3의 계산값만).
- **Android 자동 백업**: 기기 설정을 켠 경우 Google 계정으로 앱 데이터가 백업됩니다(§3 범위).
  이는 Android 시스템 기능이며 본인 계정에만 저장됩니다.

## 5. 기기 밖으로 나갈 수 있는 것 (활성화된 경우)

**① 익명 사용 통계 — TelemetryDeck (EU 호스팅)**
"온보딩을 마쳤다", "위젯을 설치했다" 같은 **발생 사실만** 보냅니다. 걸음 수·운동 시간·XP 같은
수치는 어떤 이벤트에도 싣지 않으며, 이는 자동 테스트로 강제됩니다. 기기 식별자는 복원 불가능한
해시로만 처리됩니다.

**② 오류 보고 — Sentry**
앱이 비정상 종료되면 오류 위치·기기 모델·OS 버전을 보냅니다. 건강 수치·화면 캡처는 포함되지
않으며, 오류 메시지의 숫자는 전송 전 자동 마스킹됩니다.

두 채널 모두 광고·마케팅 목적이 아니며, 제3자에게 제공되지 않습니다.

## 6. 삭제

설정 → **모든 데이터 삭제**: 성장 원장·게임 상태·설정이 이 기기에서 즉시 삭제되고 앱이 초기화
됩니다. Health Connect의 원본 건강 기록은 앱이 저장한 적이 없으므로 삭제 대상 자체가 없습니다
(원본 관리는 Health Connect 앱에서).

## 7. 아동·민감정보

- 만 14세 미만을 대상으로 하지 않습니다.
- 건강 데이터를 광고, 신용 평가, 보험, 고용 등 어떤 2차 목적에도 사용하지 않습니다.
- 앱은 의료기기가 아니며 의학적 판단에 사용할 수 없습니다.

## 9. (내부) Play Data safety 폼 매핑 — E8-5 입력

| 폼 항목 | 답 | 근거 |
|---|---|---|
| Health & fitness 수집 | Yes (앱 기능용, 기기 내 처리) | §2 — HC 읽기 |
| 수집 데이터의 전송 | **No** — 원본은 전송 안 함 | §2·§5, 코드 강제(allowlist 테스트·스크러버) |
| 공유(제3자) | No | §5 — 분석·크래시는 서비스 제공자(processor) 지위 |
| 암호화 전송 | Yes (HTTPS) | TelemetryDeck·Sentry 기본 |
| 삭제 요청 수단 | Yes — 인앱 전체 삭제 | §6, `clearApplicationUserData` |
| App functionality/Analytics | 발생 이벤트·크래시 로그(수치 없음) | §5 |
| 광고 ID | 사용 안 함 | 의존성 없음 |

**권한 화면 일치 체크리스트**: READ_STEPS/EXERCISE/HEART_RATE/SLEEP + BACKGROUND + HISTORY —
온보딩 설명 문구(§2 표와 동일 표현) = 스토어 선언 = 이 문서. 세 곳이 어긋나면 심사 반려 사유.
