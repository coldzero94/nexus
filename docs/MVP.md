# NEXUS MVP 제품 정의 v0.3 (Android 퍼스트)

기준일 2026-07-14. 원 기획서를 2인 팀 실행 범위로 좁히고 [시장 조사](./RESEARCH.md)·[벤치마크](./BENCHMARK.md)·[스택 검증](./STACK.md)을 반영한 버전. v0.2(iOS)→v0.3(Android) 변경은 §10.

## 0. 한 문장 정의

> **"체크리스트 없는 다마고치 — 당신이 움직이면 캐릭터는 알아서 자랍니다."**
> 폰·워치의 건강 데이터(걷기·러닝·근력)를 2D 캐릭터의 XP·능력치·클래스로 자동 전환하는 라이프 RPG. 캐릭터는 죽지 않고, 휴식도 성장이다.

**타깃**: 한국 20-39 자기관리에 관심 있는 스마트폰(갤럭시) 사용자. **워치는 전제가 아니라 가산점** — 있으면 더 정확한 데이터(Tier A)와 워치 경험이 열리지만, 코어 루프(수집→XP→캐릭터)는 폰 단독으로 완결된다.

**플랫폼**: Android 퍼스트 (1호 실데이터 유저가 갤럭시, Wear OS는 진짜 워치페이스 캐릭터 가능). iOS는 게이트 통과 후 확장.

## 1. 확정 원칙 (조사 근거 포함)

| 원칙 | 근거 |
|---|---|
| **자동 수집** — 수동 기록을 요구하는 단계 0개 (근력은 워치/삼성헬스 '운동 시작' 1탭까지 허용) | 카테고리 승자 전원이 자동 진영. Finch 최대 이탈 원인 = 수동 체크 피로 |
| **폰이 메인, 워치는 증폭기** — 워치 없는 사용자도 1급 타깃, 어떤 기능도 워치를 전제하지 않음 | Finch는 워치 앱 없이 ARR $30-40M (추정). 검증된 리텐션 드라이버(위젯)도 폰 표면. 개인 계수가 '자기 기준 대비'라 워치 없어도 불리하지 않음 |
| **무처벌 + 소프트 손실** — 죽음·퇴화 없음, '컨디션'만 완만히 하락(회복 가능) | 무처벌 Finch가 처벌형 Habitica의 5배 매출 (추정). Duolingo 용서 장치의 정량 효과 |
| **걸음은 재화가 아니라 몸** — 데이터는 캐릭터의 외형·능력치·히스토리에 직접 새김 | 화폐 환전형(Wokamon/Walkr) 전멸, 성장 가시화형(Pikmin/Finch) 생존 |
| **환금성 금지** — 어떤 단계에서도 현금·토큰·거래 불가 | STEPN 붕괴, 게임산업법. 비환금이 곧 무결성 전략 |
| **레이어드 설계 — 겉은 다마고치, 속은 RPG** | 한국 걷기 게임 수요의 상당분은 '귀여운 캐릭터+가벼운 플레이' |
| **"실시간" 약속 금지** — 동기화 지연(삼성헬스→HC 30~60분 추정)을 연출로 흡수 | "모험에서 돌아오는 중" 프레임. 즉시성은 나중에 워치 경로(Health Services)로만 |
| **여성 사용자 = 1급 시민** | Finch 75% 여성, 러닝 붐 동력 |
| **아트 예산은 반응 다양성에 우선** | Pokémon Sleep 이탈 원인 = 같은 애니메이션 소진 |

## 2. 화면 구성 (3탭 + 위젯)

| 표면 | 구성 | 비고 |
|---|---|---|
| **홈 (캐릭터)** | 캐릭터+표정, 컨디션 게이지, 오늘 활동 요약, 오늘 XP, 원정 상태, 다음 목표 한 줄 | 3초 안에 상태 파악 |
| **성장** | 능력치(지구력·민첩·힘·회복 / 집중·친화 잠금), 클래스 성향, 레벨/XP, 장비·배지, **XP 환산 설명 진입점** | 환산 투명성(트랭글 반면교사) |
| **프로필/설정** | Health Connect 연동 상태, 삼성헬스 연동 안내, 주간 목표, 휴식 모드, 데이터 삭제, 백업 내보내기 | 권한 거부 시 데모 모드 |
| **온보딩** | 캐릭터 생성·꾸미기(권한 전) → 권한 3종 설명 → HC 권한 → **과거 이력 소급으로 초기 레벨 부여 연출** | READ_HEALTH_DATA_HISTORY 활용 — "이미 이만큼 성장해 있었어요" |
| **홈 위젯 (1차)** | Finch 4대 장치: ① 캐릭터 상태 ② 원정 진행/완료 ③ 일일 진행바 ④ 아침/저녁 이벤트 | Glance. 잠금화면 위젯(One UI 8)은 갤럭시 실기기 검증 후 |

지도·커뮤니티 탭 없음(2단계). 워치(Wear OS)는 게이트 통과 후 — 단 3계층 설계는 [STACK.md §3](./STACK.md) 확정.

## 3. 코어 루프

1. 현실에서 걷기·러닝·근력 운동 (앱 안 켜도 됨 — 갤럭시워치/삼성헬스/폰 센서가 기록)
2. Health Connect에서 15분 주기 증분 동기화(Changes API) → 신뢰 필터 통과분만 XP 변환
3. **정산 개봉 의식**: 새 활동 반영 시(또는 앱 진입 시) XP 정산 카드
4. 능력치·클래스 성향·컨디션·표정이 변함
5. 에너지로 캐릭터를 **원정(8h)** 보냄 → 개봉 (하루 2~3회 자연 재방문 + 동기화 지연 흡수 연출)
6. 아침 "간밤의 회복 + 어제의 성장" 카드, 하루 끝 '오늘의 모험 일지'

## 4. 데이터 모델

- `ActivityRecord` — type, 시작/종료, duration, distance?, kcal?, avgHR?, `dedupKey`, **`trustTier`**, `dataOrigin`(패키지명), `recordingMethod`
- `DailySummary` / `Character`(컨디션 0-100 소프트 손실, 기분 enum 5) / `StatBlock` / `ClassAffinity`(28일 벡터)
- `RewardEvent` — 모든 XP 지급의 **불변 원장**: 멱등성 키(HC 레코드 id), provenance(dataOrigin+recordingMethod), 단조 시퀀스, 산식 버전 태그. 취소(삭제 감지 시)는 수정이 아닌 보상 이벤트 append — [BACKEND.md §1](./BACKEND.md)
- `Expedition` / `Item·Badge`(월 한정 배지 캘린더)

**신뢰 등급 (2단계 리더보드 대비 지금 박아둠)**
- Tier A: 워치 운동 세션(심박 시계열 동반) — XP 100%
- Tier B: 자동/능동 기록(온디바이스 걸음, 삼성헬스 폰 세션) — XP 70~100% (실측 후 확정)
- Tier C: 수기 입력(`RECORDING_METHOD_MANUAL_ENTRY`)·미상 서드파티 — 표시만, XP 0% (예외: 검증된 근력 로거 화이트리스트)

**무결성 최소 3종(서버 불필요)**: recordingMethod 수기 제외 / dataOrigin 신뢰 등급(⚠ 2026-06 SPN 변경 — `getCurrentDeviceDataSource()` 병용, 화이트리스트 원격 구성) / 일일 XP 인정 상한.

## 5. XP 산식 v0.3

`획득 XP = 기본 점수 × 개인 계수 × 신뢰 계수 × 균형 보너스` (일일 상한 300pt, 200pt 초과분 절반 체감)

- **기본 점수**: 걷기 1분=1pt, 러닝 1분=2pt, 근력 1분=1.5pt. 심박 시계열이 있으면 **심박존 체류 기반 ±20% 보정**(HC에 effortScore 대응물이 없어 자체 산출 — 추정치임을 UI 표기)
- **개인 계수**: 최근 14일 이동평균 대비 0.8~1.5 캡 (자기 기준 성장 — 워치 없는 유저도 공정)
- **신뢰 계수**: §4의 Tier
- **균형 보너스**: 주 4일+ 활동 & 1일+ 휴식 시 주간 정산 +15%
- **레벨 곡선**: 100×N^1.5 (치완 스프레드시트 시뮬레이션으로 검증)

### 근력 축 — 인정 경로 (워치 유무별)
| 사용자 | 경로 | Tier |
|---|---|---|
| 갤럭시워치 | 워치에서 근력 운동 시작 → 세션+심박 | A |
| 워치 없음 | 삼성헬스 폰 앱에서 '운동 시작'(능동 기록) 또는 근력 로거 앱(번핏 등) 세션 | B |
| 공통 제외 | 사후 수기 입력(MANUAL_ENTRY) | C |

힘 XP = 세션 시간 × 주당 빈도 보너스 × 심박존 보정(있을 때만). 볼륨(중량×렙) 기반 정밀 힘 스탯은 후속(로거 앱 API 제휴 시 해금). 약속 문구는 **"세트·렙 기록 강요 제로"**.

## 6. 캐릭터 시스템

v0.2와 동일 (클래스 성향 40% 규칙, 컨디션 소프트 손실, 기분 5종 트리거 표, **복귀 환영 연출 1급 기능**, 휴식 모드, 아트 스펙: 기본체 1종×표정 5+×장비 2슬롯×4~6종 + 반응 다양성 우선). 렌더링 구현은 [STACK.md §4](./STACK.md).

## 7. 수익화 (경계는 출시일에 긋는다)

- **영구 무료**: 수집·XP·성장 코어 루프 전부 (TameWatch 반면교사)
- **유료**: 꾸미기·클래스 스킨, 상세 스탯·히스토리 심화
- **광고 금지 확정** — 취향이 아니라 정책: Play는 건강 데이터 보유 앱의 광고 활용을 전면 금지
- 가격 벤치마크: 월 ₩6,900~9,900 / 연 ₩39,000~49,000 또는 일회성 병행. 가챠 없음
- 기대치: 플랜핏 누적 ~$5M (추정)이 상한 벤치마크

## 8. 성공 지표

v0.2와 동일 (D30 리텐션 중심, 위젯 설치·경유 진입, 원정 개봉률, 아침 카드 조회율, 환산 투명성) + **동기화 체감 지표**: "운동→반영" 인지 지연에 대한 불만 빈도 (원정 연출이 흡수하는지 검증).

## 10. v0.2 → v0.3 변경 요약

1. **플랫폼: iOS → Android 퍼스트** (1호 유저=갤럭시, Wear OS 워치페이스 우위, 비용). iOS 스택 검증은 git 히스토리에 보존 — 게이트 후 재사용
2. 타깃 명확화: **폰이 메인, 워치는 가산점** — 워치 없는 사용자 1급 타깃 (근력 인정 경로 표 신설)
3. HealthKit→Health Connect: wasUserEntered→recordingMethod, HKSource→dataOrigin(+SPN 변경 대응), effortScore→심박존 자체 산출
4. **"실시간 반영" 약속 금지 원칙 추가** — 삼성헬스 동기화 지연을 원정 연출로 흡수
5. 온보딩에 **과거 이력 초기 레벨 부여** 추가 (READ_HEALTH_DATA_HISTORY)
6. S0에 Play 계정·테스터 풀·실기기 실측 추가 (12명×14일 게이트 대응)
7. 잠금화면 위젯은 검증 전 약속 금지 — 홈 위젯 1차
8. 광고 금지가 "방침"에서 "플랫폼 정책상 확정"으로 승격
