# NEXUS MVP 제품 정의 v0.2

기준일 2026-07-13. 원 기획서(Project_NEXUS_제작기획서.docx)를 2인 팀 실행 범위로 좁히고, [시장 조사](./RESEARCH.md)와 [기능 벤치마크](./BENCHMARK.md)의 검증 결과를 반영한 버전. v0.1(구두 논의) 대비 변경점은 §10에 정리.

## 0. 한 문장 정의

> **"체크리스트 없는 다마고치 — 당신이 움직이면 캐릭터는 알아서 자랍니다."**
> 폰·워치의 건강 데이터(걷기·러닝·근력)를 2D 캐릭터의 XP·능력치·클래스로 자동 전환하는 라이프 RPG. 캐릭터는 죽지 않고, 휴식도 성장이다.

## 1. 확정 원칙 (조사 근거 포함)

| 원칙 | 근거 |
|---|---|
| **자동 수집** — 수동 기록을 요구하는 단계 0개 (근력은 워치 '시작' 1탭까지 허용) | 카테고리 승자 전원이 자동 진영. Finch 최대 이탈 원인 = 수동 체크 피로 |
| **무처벌 + 소프트 손실** — 죽음·퇴화 없음. 대신 '컨디션'이 완만히 하락(회복 가능) | 무처벌 Finch가 처벌형 Habitica의 5배 매출 (추정). Duolingo 용서 장치의 정량 효과. 단 잃을 게 없으면 복귀 동기도 없음 |
| **걸음은 재화가 아니라 몸** — 데이터는 캐릭터의 외형·능력치·히스토리에 직접 새김 | 화폐 환전형(Wokamon/Walkr) 전멸, 성장 가시화형(Pikmin/Finch) 생존 |
| **환금성 금지** — 어떤 단계에서도 현금·토큰·거래 불가 | STEPN -95% 붕괴, 게임산업법 환전금지. 비환금이 곧 무결성 전략 |
| **레이어드 설계 — 겉은 다마고치, 속은 RPG** | 한국 걷기 게임 수요의 상당분은 '귀여운 캐릭터+가벼운 플레이'. RPG 깊이는 온보딩 이후 서서히 |
| **폰이 본체, 워치는 얇은 창** | Pixel Pals 성공 vs 워치 전용 앱 전멸. 워치는 WidgetKit+운동 세션 중심으로 얇게 |
| **여성 사용자 = 1급 시민** | Finch 사용자 75% 여성, 국내 러닝 붐 핵심 동력이 2030 여성. 톤·캐릭터·커리큘럼 설계에 반영 |
| **아트 예산은 반응 다양성에 우선** | Pokémon Sleep 이탈 원인 = 같은 애니메이션 소진 |

## 2. 화면 구성 (3탭 + 위젯)

| 표면 | 구성 | 비고 |
|---|---|---|
| **홈 (캐릭터)** | 캐릭터+표정, 컨디션 게이지, 오늘 활동 요약, 오늘 XP, 원정 상태, 다음 목표 한 줄 | 3초 안에 상태 파악 |
| **성장** | 능력치(지구력·민첩·힘·회복 활성 / 집중·친화 잠금), 클래스 성향 게이지, 레벨/XP, 장비·배지 그리드, **XP 환산 규칙 설명 진입점** | 환산 투명성(트랭글 반면교사) |
| **프로필/설정** | HealthKit 연동 상태, 주간 목표, 휴식 모드, 데이터 삭제, 신뢰 등급 안내 | 권한 거부 시 데모 모드 |
| **온보딩** | 캐릭터 생성·꾸미기(권한 요청 전!) → 첫 XP 체험 → 권한 설명 → HealthKit 권한 → 소급 반영 연출 | Finch 순서. 계정 가입 없음 = 무기 |
| **잠금화면/홈 위젯** | Finch 4대 장치: ① 캐릭터 상태 ② 원정 진행/완료 ③ 일일 진행바 ④ 아침/저녁 마이크로 이벤트 | 위젯 = 검증된 리텐션 드라이버. 개발 자원 최우선 배분 |

지도·커뮤니티 탭 없음(후속 단계). 워치 앱은 S5+ (§9).

## 3. 코어 루프

1. 현실에서 걷기·러닝·근력 운동 (앱 안 켜도 됨)
2. HealthKit이 자동 수집 → 신뢰 필터 통과분만 XP 변환
3. **정산 개봉 의식**: 운동 종료 직후(또는 앱 진입 시) XP 정산 애니메이션
4. 능력치·클래스 성향·컨디션·표정이 변함
5. 모은 에너지로 캐릭터를 **원정(8h)** 보냄 → 몇 시간 뒤 보상 개봉 (하루 2~3회 자연 재방문)
6. 아침엔 **"간밤의 회복 + 어제의 성장 요약" 카드**, 하루 끝엔 '오늘의 모험 일지'

## 4. 데이터 모델

- `ActivityRecord` — type, 시작/종료, duration, distance?, kcal?, avgHR?, `dedupKey`, **`trustTier`(A/B/C)**, `sourceBundleId`, `effortScore?`
- `DailySummary` — 날짜, 걸음, 활동 분, 활동별 집계
- `Character` — 외형, 레벨, 누적 XP, **컨디션(0-100, 소프트 손실 대상)**, 기분(enum 5)
- `StatBlock` — 지구력/민첩/힘/회복 (+집중/친화 잠금)
- `ClassAffinity` — 최근 28일 활동 벡터 → 성향 %
- `RewardEvent` — 모든 XP 지급 로그(원본 참조 유지 → 산식 변경 시 소급 재계산)
- `Expedition` — 원정 시작/종료 시각, 보상
- `Item/Badge` — 슬롯, 해금 조건 (월 1회 한정 배지 캘린더 포함)

**신뢰 등급 (2단계 리더보드 대비 지금 박아둠)**
- Tier A: 워치 센서 세션(`com.apple.health.*` + 심박 스트림 동반) — XP 100%
- Tier B: 아이폰 걸음 — XP 70%
- Tier C: 서드파티 앱·수기 입력 — 표시만, XP 0% (예외: §5의 근력 로거 화이트리스트)

**무결성 최소 3종(서버 불필요)**: `wasUserEntered=YES` 제외 / 소스 bundle ID 화이트리스트 / 일일 XP 인정 상한. HKDevice 필드는 신뢰 판정에 사용 금지(위조 가능).

## 5. XP 산식 v0.2

`획득 XP = 기본 점수 × 개인 계수 × 신뢰 계수 × 균형 보너스` (일일 상한 300pt, 200pt 초과분 절반 체감)

- **기본 점수**: 걷기 1분=1pt, 러닝 1분=2pt, 근력 1분=1.5pt. 심박 존 데이터가 있으면 ±20% 상한 보정
- **개인 계수**: 최근 14일 이동평균 대비 달성률을 0.8~1.5로 캡 (Apple 링의 적응형 목표 원리 — 절대량이 아닌 '나의 지난 4주 대비')
- **신뢰 계수**: Tier A 1.0 / Tier B 0.7 / Tier C 0
- **균형 보너스**: 주 4일 이상 활동 + 1일 이상 휴식 시 주간 정산 +15% (휴식이 수치로 이득인 유일한 지점)
- **레벨 곡선**: 레벨 N 필요 XP = 100 × N^1.5 (스프레드시트 시뮬레이션으로 검증 예정)

### 근력(힘) 축 — 재정의 ⚠️
HealthKit에는 세트·렙·중량 타입이 없고 심박 기반 강도 추정도 저항운동에선 부정확(r=0.10~0.34). 따라서:
- **힘 XP = 세션 시간 × 주당 빈도 보너스 × workoutEffortScore 보정**(iOS 18+, 워치에서 1탭 1~10 평가)
- **플랜핏·번핏·Strong·Hevy가 HealthKit에 남긴 근력 세션을 자동 인식**(HKSource 식별)해 힘 XP 부여 — 이 사용자층(한국 20-39 리프터)은 추가 입력 0으로 온보딩됨. "기록 앱과 경쟁하지 않고 그 위에 얹힌다"
- 볼륨(중량×렙) 기반 정밀 힘 스탯은 후속(개별 앱 API 제휴 시 '고급 힘 스탯' 해금)
- 세계관 처리: 힘 스탯은 '단련의 꾸준함'을 반영 — 3축 비대칭(걷기 완전자동/러닝 준자동/근력 시작 1탭)을 숨기지 않는다
- **약속 문구**: "수동 입력 제로"(거짓) 대신 **"세트·렙 기록 강요 제로"**(100% 이행 가능)

## 6. 캐릭터 시스템

- **클래스 성향**: 최근 28일 XP 활동축 집계 → 최대 축 40% 이상이면 성향 표시(러너/전사/레인저), 미만이면 '탐색 중'. MVP 효과는 칭호+외형 코스메틱
- **컨디션(소프트 손실)**: 활동/휴식 리듬으로 완만히 오르내림. 하락해도 회복 가능, 소멸·죽음 없음. 휴식 모드 켜면 정상 보상
- **기분 5종**: {신남, 평온, 뿌듯, 심심, 휴식 중} × 트리거 표(치완 작성). **복귀 환영 연출은 1급 기능** — 오래 쉬다 돌아온 날 캐릭터가 반겨줌
- **휴식 보상**: HealthKit 수면·HRV를 로컬에서 읽어 '휴식일 = 회복 버프 이벤트'로 (Gentler Streak는 이걸 점수로만 보여줌 — 우리는 캐릭터 성장 이벤트로). 단 상한 있는 버프로 설계해 휴식이 그라인드가 되지 않게
- **아트 스펙**: 기본체 1종 × 표정 5+ × 장비 슬롯 2 × 슬롯당 4~6종. 정지 스프라이트 + 2~4프레임 idle. 조합 수 고정, 반응(대사·포즈) 다양성에 예산 우선

## 7. 수익화 (경계는 출시일에 긋는다)

- **영구 무료**: 건강 데이터 기록, XP·능력치·클래스 성장 코어 루프 전부 (TameWatch 반면교사 — 운동의 대가를 페이월 뒤에 두면 평점 2.0)
- **유료(소프트 페이월)**: 캐릭터 꾸미기·클래스 스킨, 상세 스탯·히스토리 심화, (후속) 워치 프리미엄
- **가격 벤치마크**: 월 ₩6,900~9,900 / 연 ₩39,000~49,000 구독, 또는 일회성·라이프타임 병행(서버 없는 구조와 궁합 — LifeUp/Streaks/Pixel Pals 선례). 확정은 알파 후
- **금지**: 가챠(확률 고지 의무 회피), 광고(건강 데이터 타겟팅은 Apple 정책상 원천 금지), 현금성 리워드, 무료였던 것의 후행 유료화
- **기대치**: 플랜핏 누적 ~$5M (추정)이 국내 상한 벤치마크. 목표는 볼륨이 아니라 애착 깊은 유저의 전환(전환율 현실치 2~8%)

## 8. 성공 지표

| 가설 | 지표 | 기준 |
|---|---|---|
| 캐릭터가 활동 지속을 돕는다 | **D30 리텐션** | 업계 평균 8-12%, 최상위 25% — 알파 목표는 테스터 절반이 2주차에도 주 3회+ 앱 확인 |
| 위젯이 재방문을 만든다 | 위젯 설치율, 위젯 경유 진입 비율 | Duolingo 근거로 위젯 품질에 자원 최우선 |
| 성장 규칙이 공정하게 느껴진다 | 환산 규칙 화면 조회 후 이탈률, 피드백 | 트랭글 반면교사 |
| appointment 루프가 작동한다 | 원정 개봉률, 아침 카드 조회율 | Finch/Pokémon Sleep 패턴 검증 |

게임화의 동기 효과는 수 주 내 크게 약화된다는 연구가 있으므로(BMJ/JMIR), **알파 지표가 좋아도 '커뮤니티 2막'(지역·크루) 없이는 장기 곡선이 꺾인다**는 전제로 로드맵을 유지한다.

## 10. v0.1 → v0.2 변경 요약

1. 근력→힘 스탯 "완전 자동" 폐기 → 세션 시간×빈도×effortScore + 근력 로거 얹혀가기 (HealthKit 구조 한계)
2. "조작 불가능한 인증" 표현 폐기 → 신뢰 등급 Tier A/B/C + 최소 방어 3종 데이터 모델화
3. "워치 화면에 사는 캐릭터" 약속 하향 → 컴플리케이션 스냅샷 + 운동 중 실시간 + RelevanceKit, 그리고 위젯(폰)을 S4로 상향
4. 캐릭터에 '컨디션' 소프트 손실 추가 (완전 무손실은 복귀 동기도 없앤다는 Duolingo 근거)
5. 원정(8h appointment)·아침 개봉 카드·복귀 환영 연출 신규 (Finch/Pokémon Sleep 검증 패턴)
6. 수익화 경계를 MVP 시점에 확정 (후행 유료화 금지 원칙)
7. 성공 지표를 다운로드가 아닌 D30 리텐션 중심으로
8. 여성 사용자 1급 시민 원칙 명시 (Finch 75% 여성, 러닝 붐 동력)
9. iCloud 원본 건강 데이터 동기화 금지 등 규제 체크리스트 연결 (RESEARCH.md §8)
