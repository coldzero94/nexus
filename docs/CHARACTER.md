## 1. 컨셉 한 장

**무엇인가** — 당신의 '움직임'을 양분 삼아 자라는 **작은 동반 몬스터**. **포켓몬·디지몬처럼 애착을 갖고 키워나가는 육성형 크리처**(새 형태 아님 — Finch류의 정적 마스코트와 구분). IP 없는 오리지널 캐릭터의 애착 한계는 외모가 아니라 **'능력치로 시각화된 내 기록'**에 애착을 두어 극복한다 — 걷기·러닝·근력이 이 아이의 지구력·민첩·힘·외형·기억으로 새겨지고, 쌓일수록 대체 불가능해지는 데이터 자산이 된다. 알에서 부화해 성장 단계를 밟는 육성 서사와 잘 맞는다.

**톤 — 장난기 중심 (다정 기저)** — 기본 말투는 반말·감탄사·이모지 섞인 개구쟁이. 단 진지한 순간(성취·복귀)엔 톤을 살짝 낮춰 진심을 보인다. **금지**: 잔소리, 죄책감 유발, 수치 나열식 훈계. 장난기가 표면, 다정이 바닥.

**세계관 한 문단** — 이 생물은 사용자를 대신해 세계를 조금씩 탐험하는 동행자다. 매일의 활동이 아이의 능력치로 새겨지고, 세계는 **원정**으로 열리며, 아이는 그 여정에서 이야깃거리를 물어온다. **벌은 없다** — 쉬는 날엔 같이 뒹굴고, 오래 못 봐도 돌아오면 폴짝 뛰어 반긴다. 죽지 않고, 퇴화하지 않으며, 유일한 소프트 손실은 '컨디션'뿐(회복 가능).

**애착 설계** — 온보딩 첫 3분 안에 "이 아이는 **내 운동으로만** 자란다"는 의존 관계를 각인(Finch 차용). 과거 이력 소급으로 "이미 이만큼 함께였어요" 연출.

**아트 방향** — AI 생성 스프라이트(작은 동반 몬스터, 육성형 크리처, 2D · 새 형태 지양). 스펙: 기본체 1종 × **표정 5종(기분 enum과 1:1)** × 장비 2슬롯 × 4~6종. **반응 다양성 우선**(Pokémon Sleep 이탈 원인 = 같은 애니메이션 소진). 스타일 일관성 위해 생성 시드·프롬프트 고정. *(에셋 확보는 S4 [P-3](./BACKLOG.md), 이름 후보는 별도 확정)*

## 2. 기분 판정 — 결정 트리 (우선순위)

평가기는 매 정산/앱 진입 시 아래를 **위에서부터** 평가하고 **첫 매치**로 기분을 정한다.

```
0. returning(3일+ 공백 후 첫 실행)  → [복귀 환영 씬 E4-6] → 이후 신남/뿌듯으로 진입
1. 오늘 성취 이벤트 있음             → 뿌듯
   (leveledUp OR newBadge OR weeklyGoalMet OR newRecord)
2. 오늘 활동함 (todayActiveMin > 0)
   2a. personalCoef ≥ 1.3  OR  고강도(러닝/근력 세션)  → 신남
   2b. 그 외                                          → 평온
3. 오늘 활동 안 함
   3a. restMode ON  OR  계획된 휴식(직전 리듬상 균형 휴식)  → 휴식 중
   3b. 그 외 (공백, 비휴식)                                → 심심
```

> **핵심 규칙**: 활동(2)은 휴식 모드보다 우선 — 휴식 모드여도 운동하면 신남/평온. 성취(1)는 활동 기분보다 우선.

## 3. 기분 5종 트리거 표

| 기분 | 트리거 조건 | 컨디션 영향 | 표정 | 대사 예시 (장난기 톤) |
|---|---|---|---|---|
| **뿌듯** | 오늘 성취 이벤트: `leveledUp` \| `newBadge` \| `weeklyGoalMet` \| `newRecord` | 중립 | 가슴 펴고 반짝 | "봤어 봤어?! 우리 레벨업했어 ✨" · "이 배지 내가 딴 거 맞지?! 😤" |
| **신남** | `todayActiveMin>0` **AND** (`personalCoef≥1.3` \| 고강도 세션) | 소폭 회복 | 방방 점프 | "오늘 좀 미쳤는데?! 🔥" · "헉헉— 근데 개운해 😆" |
| **평온** | `todayActiveMin>0` (위 조건 미해당, 보통 활동) | 유지·소폭 회복 | 잔잔한 미소 | "무난하고 좋았어 🙂" · "딱 적당했다 그치?" |
| **휴식 중** | 활동 없음 **AND** (`restMode ON` \| 계획된 휴식) | **하락 없음** | 이불 속 뒹굴 | "오늘은 공식 뒹굴데이 🛏️" · "쉬는 것도 훈련이라구~ 😏" |
| **심심** | 활동 없음 **AND** `restMode OFF` (공백) | 완만 하락(소프트) | 두리번·발 까딱 | "심심해~ 슬슬 나갈까? 👀" · "나 여기서 기다리는 중… 🐾" |

**복귀 환영** (E4-6, 별도 씬 · 기분 아님): `returning`=3일+ 공백 후 첫 실행 → 환영 컷 → 신남/뿌듯 진입.
대사: "왔다!! 어디 갔었어~ 보고 싶었잖아 🥹→😆". **삐짐은 1초, 반가움이 주. 처벌 없음.**

> 심심의 소프트 하락이 **유일한 손실 기제**(MVP 원칙). 캐릭터는 죽지 않는다. 심심 대사는 그리움·장난이지 잔소리가 아니다.

## 4. 평가기 입력 계약 (E4-4)

| 변수 | 타입 | 의미 |
|---|---|---|
| `daysSinceActivity` | int | 마지막 활동 이후 경과일 (0=오늘 활동) |
| `restMode` | bool | 사용자 휴식 모드 ON |
| `todayActiveMin` | int | 오늘 총 활동 분 |
| `personalCoef` | float(0.8–1.5) | 오늘 개인계수 (XP 산식) |
| `leveledUp` `newBadge` `weeklyGoalMet` `newRecord` | bool | 오늘 정산 성취 이벤트 |
| `condition` | int(0–100) | 현재 컨디션 |
| `returning` | bool | 3일+ 공백 후 첫 실행 (복귀 씬) |
| `timeOfDay` | enum(아침/낮/저녁) | 대사 변주용(선택) |

규칙은 데이터 파일 [`assets/mood_triggers_v1.json`](./assets/mood_triggers_v1.json)로도 제공 — **표만 고치면 코드 수정 없이 반영**(E4-4 완료 기준).
