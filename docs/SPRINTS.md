# NEXUS 스프린트 계획

- **주기**: 2주 스프린트 (S0만 1주). 스프린트 종료일 = TestFlight 데모 빌드가 치완 폰에 도착하는 날.
- **"완료"의 정의**: 코드 머지가 아니라 **만든 사람이 아닌 다른 한 명이 자기 폰에서 확인하고 코멘트를 남긴 것**.
- **개발은 둘 다 한다** — 아래 담당 표기는 리드일 뿐, 구현 이슈는 플래닝에서 나눠 가진다.
- **GitHub에서 보는 법**: 스프린트 = [마일스톤](https://github.com/coldzero94/nexus/milestones) (진행률 바 제공), 작업 현황 = 프로젝트 보드.
- 제품 스펙 근거는 [MVP.md](./MVP.md), 설계 근거는 [BENCHMARK.md](./BENCHMARK.md) 참고.

## S1 · "데이터가 흐른다" (7/21 – 8/3)

| # | 작업 | 완료 기준 |
|---|---|---|
| 1 | 온보딩 v0: 캐릭터 생성(임시 그래픽) → 권한 설명 → HealthKit 권한 요청 | 권한 거부해도 앱 사용 가능(데모 모드) |
| 2 | 걸음 읽기 + 일별 집계(`DailySummary`) | 건강 앱 수치와 일치(오차는 원인 설명 가능) |
| 3 | HKWorkout 세션 읽기 + 중복 제거(`dedupKey`) | 워치·폰 중복 기록이 1건으로 합쳐짐 |
| 4 | 신뢰 필터 3종: `wasUserEntered` 제외, 소스 화이트리스트, 일일 상한 | 단축어로 주입한 가짜 걸음이 XP 대상에서 제외됨 |
| 5 | 활동 리스트 화면 (최근 7일) | 치완 실데이터 QA 통과 |

**치완**: 자기 폰·워치 데이터로 대조 QA + 버그 리포트 (HealthKit 데이터는 기기별 편차·지연·중복이 지저분함 — 이 검증이 S1의 절반).

## 스프린트 의식

- **플래닝 (스프린트 첫날, 30분)**: 마일스톤에서 이슈 확정, 서로의 이번 스프린트 약속 1줄씩
- **주간 싱크 (매주 1회, 30분)**: 진행 확인 + 막힌 것 + 결정 필요한 것
- **데모 & 회고 (마지막 날, 30분)**: TestFlight 빌드로 데모 → 잘된 것/막힌 것/다음에 바꿀 것 각 1개
- 나머지는 전부 비동기: 이슈 코멘트 + 카톡
