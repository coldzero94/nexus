# NEXUS 스프린트 계획 (Android)

- **주기**: 2주 스프린트 (S0만 1주). 스프린트 종료일 = 새 빌드가 **Firebase App Distribution으로 서로의 폰에 도착**하는 날.
- **"완료"의 정의**: 코드 머지가 아니라 **만든 사람이 아닌 다른 한 명이 자기 기기에서 확인하고 코멘트를 남긴 것**.
- **개발은 둘 다 한다** — 담당 표기는 리드일 뿐, 구현 이슈는 플래닝에서 나눠 가진다.
- **GitHub에서 보는 법**: 스프린트 = [마일스톤](https://github.com/coldzero94/nexus/milestones), 현황 = [보드](https://github.com/users/coldzero94/projects/1).
- 태스크 상세는 [BACKLOG.md](./BACKLOG.md), 스택은 [STACK.md](./STACK.md).

## 일정 요약

| 스프린트 | 기간 | 목표 (데모에서 보이는 것) |

**게이트**: 알파 D14에서 테스터 절반 이상이 주 3회+ 앱을 열면 → 워치 착수. 못 넘으면 코어 루프를 고친다. 커뮤니티는 그다음.

**Play 타임라인 주의**: 신규 개인 계정은 공개 출시 전 "비공개 테스트 12명×14일" + Health Connect 선언 심사가 필수라, S0의 계정 생성과 S4의 closed test 개시가 늦어지면 공개 출시가 그대로 밀린다 (개발 완료 후 공개 출시까지 4~6주 버퍼).

## S1 · "데이터가 흐른다" (7/21 – 8/3) — [BACKLOG E2](./BACKLOG.md)

권한 3종 플로우 / aggregate 걸음 / ExerciseSession 3축 / Changes 증분+WorkManager / 신뢰 필터(recordingMethod·dataOrigin·상한) / 삭제→보상 이벤트 / Room v1 / 과거 이력 초기 레벨 / 활동 리스트.
**데모**: 치완 갤럭시에서 최근 7일 걸음·운동이 삼성헬스 수치와 일치. **치완**: 실데이터 교차 QA + 버그 이슈 (삼성헬스→HC 동기화가 지저분한 게 정상 — 이 검증이 S1의 절반).

## S2 · "성장이 보인다" (8/4 – 8/17) — [BACKLOG E3](./BACKLOG.md)

## S3 · "캐릭터가 산다" (8/18 – 8/31) — [BACKLOG E4](./BACKLOG.md)

## S4 · "알파" (9/1 – 9/14) — [BACKLOG E5~E8](./BACKLOG.md)
